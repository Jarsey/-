命名

  1. 命名应该显示出你的意图
  2. 函数通常是动词，而类通常是名词
  3. 变量、函数和类的命名应该能三个回答：它为什么存在，它是做什么的，它用在哪里

函数

  1. “Functions should DO ONLY ONE THING.”
  2. 函数应该尽可能短小
  3. 函数应该只做一件事情
  4. 函数不应该出现嵌套的结构
  5. 函数中的语句都处于同一抽象级别

注释

  1. 少量简介，清晰明了
  2. 要随着代码的更新而更新

格式化代码很重要

错误处理

  1. 代码应该整洁、健壮，并且以一种优雅的方式处理错误
  2. 在代码中使用try-catch-finally可能是一种比较好的选择。它能帮助你在try部分定义你希望执行的代码，而不用去担心代码出错时怎么办。
